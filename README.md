# kvdb-pong-matchmaker

A pong game where:

1. There is a matching engine built on kvdb.io, the user can select someone else to play with and then the game starts, and they are removed from the match making lobby

2. the game state is constantly written and read to kvdb.io

Start with laying out the plan for how to use kvdb, namespaces etc

Use this public kvdb bucket: CFh5eziuGXnwmYdXC6iiEt

## Setup

```sh
git clone https://github.com/GPT-Engineer-App/kvdb-pong-matchmaker.git
cd kvdb-pong-matchmaker
npm i
```

```sh
npm run dev
```

This will run a dev server with auto reloading and an instant preview.
